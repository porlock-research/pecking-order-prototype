Cartridge Lifecycle: Where do results live?

The core problem: cartridges vanish after completion. Players never see rich results.
Two places need to show cartridge state — the chat view (live) and the schedule view (persistent).

Which rendering approach for the chat view?

The active cartridge and result announcement need to live in the chat timeline

A

GM Messages Only

Cartridge interactions appear as Game Master chat messages. Announcement = GM message. Active state = pinned GM card with inline buttons. Results = GM message with detail. No separate cartridge panel — everything lives in chat.

Pros

  • Natural chat flow
  • No mode switching
  • Results persist in chat history

Cons

  • Complex cartridges (arcade games) may not fit in chat
  • Pinning gets complex
B

GM Announce + Cartridge Panel + GM Results

GM message announces the mechanic. Active cartridge uses existing panel system (full UI). When complete, GM message announces results with rich detail. Best of both: narrative framing + full interactivity.

Pros

  • Rich cartridge UIs preserved
  • Narrative bookends (GM intro + GM results)
  • Minimal change to existing panels

Cons

  • Mode switch still exists (chat → panel → chat)
  • Need to coordinate GM messages with panel lifecycle
C

Hybrid: Simple in chat, complex in panel

Dilemmas and voting (simple decisions) render inline in chat as GM cards. Arcade games and complex cartridges use panels. Results always appear as GM messages regardless.

Pros

  • Right tool for the job
  • Simple mechanics feel lightweight
  • Complex mechanics get full UI

Cons

  • Two rendering paths to maintain
  • Inconsistent UX between cartridge types
