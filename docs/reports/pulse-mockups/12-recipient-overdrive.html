Pulse — Recipient Overdrive (sketch)

Sender bursts are a 0.95s dopamine echo (peak → float → fade). Received bursts anchor the sender's face for ~2.1s: anticipation (portrait scales in + gold edge) → hold (portrait + amount read clearly) → exit. Tap any button to fire a burst inside the phone.

Notes · 2026-04-17

  • Proposal under evaluation. Received bursts are not yet wired in code — this is a visual-only sketch to evaluate direction. Existing SilverBurst.tsx / NudgeBurst.tsx fire only from sender-side dispatch.
  • Three asymmetries between sender and recipient: (1) anticipation beat so it doesn't feel like a jump-scare on app open, (2) longer hold for comprehension, (3) face as anchor instead of text-only.
  • Coalescing when offline queue > 3: one summary burst with up to 3 portraits + total amount + "from N friends." Prevents a 15-second confetti marathon on return.
  • Dispatcher adds a direction: 'sent' | 'received' discriminator to the CustomEvent detail. Burst components branch on direction; existing sender path untouched.
  • Stacking in real app: 800ms gap between queued bursts so each registers as its own event.
