Pulse Reveal Redesign — Mockups

Mockups for three surfaces: the elimination overlay (dramatic interlude on push-notified game load), the winner overlay (euphoric finale), and the new mechanism-aware chat narrator card that renders inline in the Pulse chat feed when an elimination or winner event fires. PhaseTransition is out of scope. Persona images and theme tokens pulled from the live Pulse shell.

Fonts: Clash Display (display) + Satoshi (body) · Icons: Phosphor Fill · Palette: Pulse OKLCH tokens

1. Elimination Reveal

Peak — gut punch

Full-bleed persona photo. Medium pacing for an async game: photo enters in colour, holds ~400ms, grayscale drains over ~1.6s. Narrator line fades up at +900ms, mechanism subtitle at +1300ms. Full settle ~2s. Tap anywhere to dismiss, no auto-timeout. The goal is drama, not detail — the voting cartridge's VotingResultHero handles tally + who-voted-whom when the player taps in for specifics.

3. Chat narrator card (bolder)

Persistent · in‑feed

Currently, Pulse's ChatView.tsx only renders SOCIAL_INVITE ticker events as narrator lines. Elimination / winner tickers are dropped on the floor. This adds a mechanism-aware chat card for those events — bigger beat than the quiet NarratorLine above, but still inline in the scrolling feed (no modals). Grammar: accent-tinted hairlines top & bottom (echoing NarratorLine's gradient hairlines at louder scale), persona photo as anchor (desaturated for elim, gold-framed for winner), Phosphor icon + tracked mechanism kicker, big Clash Display name, italic Satoshi subtitle from VOTE_TYPE_INFO.eliminatedSubtitle. Persistent on scrollback — days later it still reads as the day's event marker.

Notes for the implementing agent:
· Scope: elimination overlay + winner overlay + chat narrator card. PhaseTransition is out of scope for this pass.
· Fonts: Clash Display (display) + Satoshi (body) — imported via FontShare in pulse-theme.css. Use --po-font-display / --po-font-body at call sites.
· Icons: Phosphor Fill via ./icons.ts — re-export Crown + the per-mechanism icons (Scales, HandFist, Drop, Trophy, HourglassLow, ShieldSlash, HeartBreak, Sword).
· Pacing (overlay): medium — photo enters in colour, holds ~400ms, grayscale drains over ~1.6s. Narrator line ~900ms, mechanism subtitle ~1300ms. Full settle ~2s. No auto-dismiss.
· Reduced motion: cross-fade only. Jump to final grayscale state, no crown bounce, no confetti, no chat-card slide-in.
· Mechanism awareness (overlay & card): use VOTE_TYPE_INFO.revealLabel and VOTE_TYPE_INFO.eliminatedSubtitle (already shipped in packages/shared-types/src/vote-type-info.ts). Card accent from VOTE_ACCENT.
· Overlay narrator line: dramatic sentence (e.g. "Pack your bags, Daisy.") from a per-mechanism line pool; selection deterministic via hash of (gameId, dayIndex, eliminatedId) so catch-up replay is stable.
· Chat card: renders when a ELIMINATION or PHASE_WINNER ticker arrives. Update ChatView.tsx filter to include these categories alongside SOCIAL_INVITE.
· View Transitions API for overlay→card morph: tag both portraits with view-transition-name: elim-portrait-${playerId}; wrap dismiss in document.startViewTransition. Chat card must be mounted before dismiss. Feature-detect; fall back to framer-motion cross-fade. Chrome 111+, Safari 18+. Same pattern for winner-portrait-${winnerId}.
